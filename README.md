# okinawa-lodging-tax

沖縄県宿泊税の税額算定・集計・帳簿保存・領収書発行・申告用データ出力を行うアプリ。

**1施設=1インスタンスの汎用アプリ**として設計する。
クラウド型マルチテナントSaaSではなく、事業者ごとに自分のインスタンスを持つ形態を志向する。

## 起点と方向性

- **第一の顧客は本リポジトリ運営会社の自社民泊**（ドッグフーディング前提）
  - アプリ v0 の検証に加え、将来販売する **Mac / iPad / Square キット構成そのもの** も自社民泊で検証する。キッティング手順・業務フロー・スタッフ教育負荷・端末トラブル時のリカバリ運用までを洗い出し、v1 の助成金キット販売モデル具体化の土台にする
- まず自社の民泊で日常運用できるミニマム版（v0）を作る
- v0で得た知見を元に、他の小規模宿泊事業者へ横展開できる汎用アプリへ育てる
- 沖縄県以外の宿泊税自治体（京都・東京・倶知安・福岡等）へは「税ルールプラグイン追加」で対応する想定
- **配布モデル**: 沖縄北部を中心に小規模宿泊事業者の端末保持率の低さが宿泊税導入の地域課題になっているため、将来的には **県の宿泊税関連の助成金制度（制度名・要件は今後確認）を活用し、okinawa-lodging-tax をプリインストールした Mac / iPad / Square 同梱の端末キットとして販売する** ことを志向する。ただしアプリ自体は Docker 運用で OS 非依存を維持し、既に PC と紙台帳で運用している事業者もそのまま使える状態を崩さない。
- **紙台帳運用と iPad 現場入力運用のどちらも選べる**ことを v0 から担保する。iPad は必須ではなくオプションの現場入力デバイスとして対応する。

### v0 の運用想定

- 利用ユーザー: スタッフ2〜3人（同社内）
- ロール: 全員同権限（月次集計・エクスポート・申告データ出力も全員可）
- 対象施設: 1施設
- 利用頻度: 日次入力（紙台帳から転記）と月次の申告作業（集計確認・エクスポート）を前提

### v0でやらないこと

設定UI画面、OTA連携、宿泊者名簿提出、マルチ施設対応、ロールベース権限、SSO・多要素認証、本格的な監査ログ、電帳法の本格対応。
詳細は [v0要件定義](./docs/v0-requirements.md) 参照。

## 設計原則（汎用化のための3規律）

v0時点から守る規律。これにより v0の小ささと将来の汎用化を両立する。

1. **ユーザー固有データはコードに書かない** — テキスト設定は Rails credentials、バイナリは `app/assets/images/facility/*` に分離管理（[decisions.md 2026-04-07](./docs/decisions.md) 参照）
2. **税ロジックを自治体ごとに分離** — `app/services/tax_rules/<prefecture>.rb`、`tax_rule_version` で条例改定を吸収
3. **領収書・帳票はテンプレート化** — ロゴ・住所等は設定から差し込み

## ロードマップ（概略）

| フェーズ | ゴール |
|---|---|
| **v0** | 自社民泊で2〜3人スタッフが日常運用できるミニマム版（動作要件は OS 非依存・推奨 Mac、iPad 現場入力はオプション対応、Square は決済端末として前提化） |
| **v1** | 他の小規模宿泊事業者にもインストールして使えるレベル（OTA連携・宿泊者名簿提出・**Square API 連携**・**助成金活用キット販売モデルの具体化**：キッティング手順 / 助成金申請資料 / 初期セットアップ自動化） |
| **v1.x+** | 多自治体対応、配布の仕組み、ロール権限・SSO・本格的な監査ログ |

クラウド型マルチテナントSaaSは現時点では志向しない。

## 技術スタック方針

メインスタックは **Ruby on Rails**（独立Railsアプリとして構築）。
`landbase_ai_suite` の技術スタック・開発規約を踏襲する。詳細は [docs/tech-stack.md](./docs/tech-stack.md) 参照。

## LP（ランディングページ）

`lp/` ディレクトリに静的 HTML として配置。Tailwind CSS v4 をローカルビルドで使用。

```bash
cd lp
npm install        # 初回のみ
npm run build      # dist/output.css を生成
npm run watch      # 開発時（ファイル変更で自動リビルド）
```
