# okinawa-lodging-tax

沖縄県宿泊税の税額算定・集計・帳簿保存・領収書発行・申告用データ出力を行うアプリ。

**1施設=1インスタンスの汎用アプリ**として設計する。
クラウド型マルチテナントSaaSではなく、オーナーごとに自分のインスタンスを持つ形態を志向する。

## 起点と方向性

- **第一の顧客は本リポジトリのオーナー自身**（小規模民泊を運営）
- まず自分の民泊で日常運用できるミニマム版（v0）を作る
- v0で得た知見を元に、他の小規模宿泊事業者へ横展開できる汎用アプリへ育てる
- 沖縄県以外の宿泊税自治体（京都・東京・倶知安・福岡等）へは「税ルールプラグイン追加」で対応する想定

## v0スコープ

| # | 機能 |
|---|---|
| 1 | 宿泊実績の手入力（紙台帳から転記） |
| 2 | 宿泊税の自動計算 |
| 3 | 月次集計 |
| 4 | 領収書PDF出力（宿泊料金と宿泊税を分離表示） |
| 5 | 月次データのエクスポート（申告書転記用） |

### v0でやらないこと

認証、設定UI画面、OTA連携、宿泊者名簿提出、マルチ施設対応、本格的な監査ログ・電帳法対応。
詳細は [v0要件定義](./docs/v0-requirements.md) 参照。

## 設計原則（汎用化のための3規律）

v0時点から守る規律。これにより v0の小ささと将来の汎用化を両立する。

1. **ユーザー固有データはコードに書かない** — `config/facility.json` に集約
2. **税ロジックを自治体ごとに分離** — `src/tax-rules/<prefecture>.ts`
3. **領収書・帳票はテンプレート化** — ロゴ・住所等は設定から差し込み

## ロードマップ（概略）

| フェーズ | ゴール |
|---|---|
| **v0** | 自分の民泊で日常運用できるミニマム版 |
| **v1** | 他の小規模宿泊事業者にもインストールして使えるレベル（OTA連携・宿泊者名簿提出等） |
| **v1.x+** | 多自治体対応、配布の仕組み、必要に応じて認証・監査ログ強化 |

クラウド型マルチテナントSaaSは現時点では志向しない。
