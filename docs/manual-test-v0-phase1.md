# 統合テスト手順書 — v0 Phase 1（User / Stay / 税計算）

## 前提

- `make up` でコンテナが起動していること
- http://localhost:3000 にアクセスできること

---

## 1. テストユーザーの作成

コンテナ内でテストユーザーを作成する（未作成の場合）:

```bash
make bash
rails runner "User.find_or_create_by!(email: 'admin@example.com') { |u| u.name = 'Admin'; u.password = 'password123' }"
exit
```

---

## 2. 認証基盤

### 2-1. 未認証アクセスのリダイレクト

| # | 操作 | 期待結果 |

## 3. ユーザー管理

### 3-1. ユーザー追加

| # | 操作 | 期待結果 |

| 1 | ナビゲーションの「ユーザー管理」をクリック | ユーザー一覧が表示される |

### 3-2. ユーザー編集

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | 「スタッフA」の「編集」をクリック | 編集フォームが表示される |
| 2 | 名前を「スタッフB」に変更して更新 | 一覧の名前が「スタッフB」に変わる |

### 3-3. ユーザー無効化・有効化

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | 「スタッフB」の「無効化」をクリック → 確認ダイアログで「OK」 | 状態が「無効」に変わる、行がグレーアウト |

| 3 | `admin@example.com` で再ログインし、「スタッフB」を「有効化」 | 状態が「有効」に戻る |
| 4 | 再度 `staff-a@example.com` でログイン | ログインできる |

## 4. 宿泊実績の登録

### 4-1. 通常ケース

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | 「宿泊実績」→「新規登録」をクリック | 入力フォームが表示される |
| 2 | 以下を入力して「登録」: | |
|   | チェックイン日: `2027-04-01` | |
|   | 泊数: `2` | |
|   | 代表者名: `山田 太郎` | |
|   | 総人数: `3`、課税対象: `3`、免除: `0` | |
|   | 1人1泊料金: `8000` | |
|   | チャネル: `Airbnb` | |
|   | 決済手段: `Square` | |
| 3 | 一覧に戻る | 登録した宿泊実績が表示される |
| 4 | 税額欄を確認 | **960円** と表示される（8,000 × 2% × 2泊 × 3人 = 960） |

### 4-2. 上限超過ケース

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | 新規登録: 1人1泊料金 `150000`、1泊、1人 | |
| 2 | 税額を確認 | **2,000円** と表示される（上限10万円 × 2% = 2,000） |

### 4-3. 端数発生ケース

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | 新規登録: 1人1泊料金 `3333`、1泊、1人 | |
| 2 | 税額を確認 | **66円** と表示される（3,333 × 2% = 66.66 → 切捨 66） |

### 4-4. 免除人数ありケース

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | 新規登録: 総人数 `3`、課税対象 `2`、免除 `1`、料金 `10000`、1泊 | |
| 2 | 税額を確認 | **400円** と表示される（10,000 × 2% × 1泊 × **2人** = 400） |

### 4-5. バリデーション

| # | 操作 | 期待結果 |
|---|------|---------|
| 1 | 代表者名を空にして登録 | バリデーションエラーが表示される |
| 2 | 総人数 `3`、課税対象 `2`、免除 `0` で登録 | 「合計と一致する必要があります」エラー |
